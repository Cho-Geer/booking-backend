# 数据库配置规范

## 概述
根据项目需求文档和技术栈更新说明，预约系统应使用PostgreSQL 16+数据库配合Prisma 5+ ORM进行开发。本文档详细说明了正确的数据库配置方案，以替代当前错误的MySQL + TypeORM配置。

## 当前错误配置分析

### 错误的技术栈
- **数据库**: MySQL (实际使用)
- **ORM框架**: TypeORM (实际使用)

### 错误的配置文件
在`booking-backend/src/app.module.ts`中存在以下错误配置：
```typescript
TypeOrmModule.forRoot({
  type: 'mysql',  // 错误：应为PostgreSQL
  host: process.env.DB_HOST || 'localhost',
  port: parseInt(process.env.DB_PORT) || 3306,  // MySQL默认端口
  username: process.env.DB_USERNAME || 'root',
  password: process.env.DB_PASSWORD || 'password',
  database: process.env.DB_NAME || 'booking_system',
  entities: [__dirname + '/**/*.entity{.ts,.js}'],
  synchronize: process.env.NODE_ENV === 'development',
  logging: process.env.NODE_ENV === 'development',
})
```

### 存在的问题
1. 使用了错误的数据库类型（MySQL而非PostgreSQL）
2. 使用了错误的ORM框架（TypeORM而非Prisma）
3. 数据库端口配置为MySQL默认端口（3306）
4. 未利用PostgreSQL的高级特性

## 正确的数据库配置要求

### 技术栈要求
根据`READ.md`文档中的技术要求：
- **数据库**: PostgreSQL 16+ (最新稳定版本)
- **ORM**: Prisma 5+ (现代化ORM解决方案)

### 数据模型设计
根据`数据模型设计.md`文档，数据模型基于PostgreSQL最佳实践设计，包含：
1. **用户相关模型**: users表、user_sessions表
2. **预约相关模型**: time_slots表、appointments表、appointment_history表
3. **系统配置模型**: system_settings表、blocked_time_slots表
4. **通知和日志模型**: notifications表、activity_logs表
5. **统计和报表模型**: appointment_statistics表

### PostgreSQL特性要求
数据模型设计中使用了PostgreSQL特有的功能：
1. **UUID数据类型**: 使用`gen_random_uuid()`生成主键
2. **INET数据类型**: 存储IP地址
3. **JSONB数据类型**: 存储结构化数据
4. **时间戳默认值**: 使用`CURRENT_TIMESTAMP`

### 3. Prisma配置
创建Prisma配置文件`prisma/schema.prisma`：
```prisma
// datasource.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// 用户模型
model User {
  id                String    @id @default(uuid()) @db.Uuid
  name              String    @db.VarChar(100)
  phone             String    @unique @db.VarChar(20)
  phoneHash         String    @unique @map("phone_hash") @db.VarChar(64)
  email             String?   @db.VarChar(255)
  wechat            String?   @db.VarChar(100)
  verificationCode  String?   @map("verification_code") @db.VarChar(6)
  codeExpiresAt     DateTime? @map("code_expires_at")
  isVerified        Boolean   @default(false) @map("is_verified")
  userType          UserType  @default(CUSTOMER) @map("user_type")
  status            UserStatus @default(ACTIVE)
  lastLoginAt       DateTime? @map("last_login_at")
  loginCount        Int       @default(0) @map("login_count")
  ipAddress         String?   @map("ip_address") @db.Inet
  userAgent         String?   @map("user_agent") @db.Text
  createdAt         DateTime  @default(now()) @map("created_at")
  updatedAt         DateTime  @updatedAt @map("updated_at")
  sessions          UserSession[]
  appointments      Appointment[]
  blockedSlots      BlockedTimeSlot[]
  notifications     Notification[]
  activityLogs      ActivityLog[]
  appointmentHistory AppointmentHistory[]
  systemLogs        SystemLog[]
  
  @@index([phoneHash])
  @@index([status])
  @@index([createdAt])
  @@map("users")
}

// 用户会话模型
model UserSession {
  id                   String    @id @default(uuid()) @db.Uuid
  userId               String    @map("user_id") @db.Uuid
  sessionToken         String    @unique @map("session_token") @db.VarChar(255)
  refreshToken         String?   @unique @map("refresh_token") @db.VarChar(255)
  expiresAt            DateTime  @map("expires_at")
  refreshExpiresAt     DateTime? @map("refresh_expires_at")
  ipAddress            String?   @map("ip_address") @db.Inet
  userAgent            String?   @map("user_agent") @db.Text
  deviceInfo           Json?     @map("device_info") @db.JsonB
  isActive             Boolean   @default(true) @map("is_active")
  createdAt            DateTime  @default(now()) @map("created_at")

  user                 User      @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId])
  @@index([sessionToken])
  @@index([expiresAt])
  @@map("user_sessions")
}

// 时间段配置模型
model TimeSlot {
  id              String   @id @default(uuid()) @db.Uuid
  slotTime        String   @db.VarChar(8) // HH:mm:ss格式
  durationMinutes Int      @default(30)
  isActive        Boolean  @default(true)
  displayOrder    Int?
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  
  // 关联关系
  appointments    Appointment[]
  blockedSlots    BlockedTimeSlot[]
  
  @@unique([slotTime])
  @@index([isActive])
  @@index([displayOrder])
  @@map("time_slots")
}

// 预约记录模型
model Appointment {
  id                  String            @id @default(uuid()) @db.Uuid
  appointmentNumber   String            @unique @db.VarChar(20) // AP-yyyymmdd-0001格式
  userId              String?           @db.Uuid
  appointmentDate     DateTime          @db.Date
  timeSlotId          String            @db.Uuid
  status              AppointmentStatus @default(PENDING)
  
  // 用户信息快照
  customerName        String            @db.VarChar(100)
  customerPhone       String            @db.VarChar(20) // 加密存储
  customerEmail       String?           @db.VarChar(255)
  customerWechat      String?           @db.VarChar(100)
  notes               String?
  
  // 系统信息
  ipAddress           String?           @db.Inet
  userAgent           String?
  confirmationSent    Boolean           @default(false)
  reminderSent        Boolean           @default(false)
  
  // 时间记录
  createdAt           DateTime          @default(now())
  updatedAt           DateTime          @updatedAt
  confirmedAt         DateTime?
  cancelledAt         DateTime?
  completedAt         DateTime?
  
  // 关联关系
  user                User?             @relation(fields: [userId], references: [id])
  timeSlot            TimeSlot          @relation(fields: [timeSlotId], references: [id])
  history             AppointmentHistory[]
  notifications       Notification[]
  
  @@index([userId])
  @@index([appointmentDate])
  @@index([timeSlotId])
  @@index([status])
  @@index([appointmentNumber])
  @@map("appointments")
}

// 预约历史记录模型
model AppointmentHistory {
  id               String              @id @default(uuid()) @db.Uuid
  appointmentId    String              @map("appointment_id") @db.Uuid
  action           String              @db.VarChar(50)
  previousStatus   String?             @map("previous_status") @db.VarChar(20)
  newStatus        String?             @map("new_status") @db.VarChar(20)
  changedBy        String?             @map("changed_by") @db.Uuid
  changeReason     String?             @map("change_reason") @db.Text
  metadata         Json?               @db.JsonB
  createdAt        DateTime            @default(now()) @map("created_at")

  appointment      Appointment         @relation(fields: [appointmentId], references: [id], onDelete: Cascade)
  user             User?               @relation(fields: [changedBy], references: [id])

  @@index([appointmentId])
  @@index([createdAt])
  @@map("appointment_history")
}

// 系统配置模型
model SystemSetting {
  id            String         @id @default(uuid()) @db.Uuid
  settingKey    String         @unique @map("setting_key") @db.VarChar(100)
  settingValue  String?        @map("setting_value") @db.Text
  settingType   SettingType    @default(STRING) @map("setting_type")
  description   String?        @db.Text
  isPublic      Boolean        @default(false) @map("is_public")
  category      SettingCategory @default(GENERAL)
  createdAt     DateTime       @default(now()) @map("created_at")
  updatedAt     DateTime       @updatedAt @map("updated_at")

  @@map("system_settings")
}

// 不可用时间段模型
model BlockedTimeSlot {
  id          String     @id @default(uuid()) @db.Uuid
  date        DateTime   @db.Date
  startTime   String     @map("start_time") @db.VarChar(8) // HH:mm:ss格式
  endTime     String     @map("end_time") @db.VarChar(8)   // HH:mm:ss格式
  reason      String?    @db.VarChar(255)
  isRecurring Boolean    @default(false) @map("is_recurring")
  createdAt   DateTime   @default(now()) @map("created_at")
  updatedAt   DateTime   @updatedAt @map("updated_at")

  @@unique([date, startTime, endTime])
  @@map("blocked_time_slots")
}

// 通知记录模型
model Notification {
  id            String              @id @default(uuid()) @db.Uuid
  userId        String              @map("user_id") @db.Uuid
  appointmentId String?             @map("appointment_id") @db.Uuid
  type          NotificationType
  title         String              @db.VarChar(200)
  content       String              @db.Text
  isRead        Boolean             @default(false) @map("is_read")
  status        NotificationStatus  @default(PENDING)
  scheduledFor  DateTime?           @map("scheduled_for")
  sentAt        DateTime?           @map("sent_at")
  readAt        DateTime?           @map("read_at")
  metadata      Json?               @db.JsonB
  createdAt     DateTime            @default(now()) @map("created_at")

  user          User                @relation(fields: [userId], references: [id], onDelete: Cascade)
  appointment   Appointment?        @relation(fields: [appointmentId], references: [id], onDelete: Cascade)

  @@index([userId])
  @@index([appointmentId])
  @@index([status])
  @@index([createdAt])
  @@map("notifications")
}
}

// 操作日志模型
model ActivityLog {
  id            String    @id @default(uuid()) @db.Uuid
  userId        String?   @map("user_id") @db.Uuid
  action        String    @db.VarChar(100)
  resourceType  String?   @map("resource_type") @db.VarChar(50)
  resourceId    String?   @map("resource_id") @db.Uuid
  ipAddress     String?   @map("ip_address") @db.Inet
  userAgent     String?   @map("user_agent") @db.Text
  createdAt     DateTime  @default(now()) @map("created_at")

  user          User?     @relation(fields: [userId], references: [id])

  @@index([userId])
  @@index([createdAt])
  @@index([resourceType, resourceId])
  @@map("activity_logs")
}

// 系统日志模型
model SystemLog {
  id            String    @id @default(uuid()) @db.Uuid
  level         String    @db.VarChar(20)
  message       String    @db.Text
  context       Json?     @db.JsonB
  userId        String?   @map("user_id") @db.Uuid
  ipAddress     String?   @map("ip_address") @db.VarChar(45)
  userAgent     String?   @map("user_agent") @db.Text
  createdAt     DateTime  @default(now()) @map("created_at")

  user          User?     @relation(fields: [userId], references: [id])

  @@index([level])
  @@index([createdAt])
  @@index([userId])
  @@map("system_logs")
}

// 预约统计模型
model AppointmentStatistic {
  id                     String   @id @default(uuid()) @db.Uuid
  statDate               DateTime @unique @db.Date
  totalAppointments      Int      @default(0) @map("total_appointments")
  confirmedAppointments  Int      @default(0) @map("confirmed_appointments")
  cancelledAppointments  Int      @default(0) @map("cancelled_appointments")
  completedAppointments  Int      @default(0) @map("completed_appointments")
  newUsers               Int      @default(0)
  returningUsers         Int      @default(0)
  peakHour               String?  @db.VarChar(5)
  noShowCount            Int      @default(0) @map("no_show_count")
  averageLeadTimeHours   Float?   @map("average_lead_time_hours")
  createdAt              DateTime @default(now()) @map("created_at")
  updatedAt              DateTime @updatedAt @map("updated_at")
  
  @@index([statDate])
  @@map("appointment_statistics")
}
```

### 5. Prisma服务实现
创建Prisma服务`booking-backend/src/modules/prisma/prisma.service.ts`：
```typescript
import { Injectable, OnModuleInit, OnModuleDestroy } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

/**
 * Prisma服务类
 * 提供数据库连接和ORM功能
 */
@Injectable()
export class PrismaService extends PrismaClient implements OnModuleInit, OnModuleDestroy {
  /**
   * 应用启动时连接数据库
   */
  async onModuleInit() {
    await this.$connect();
  }

  /**
   * 应用关闭时断开数据库连接
   */
  async onModuleDestroy() {
    await this.$disconnect();
  }
}
```

创建Prisma模块`booking-backend/src/modules/prisma/prisma.module.ts`：
```typescript
import { Module } from '@nestjs/common';
import { PrismaService } from './prisma.service';

/**
 * Prisma模块
 * 提供Prisma服务的全局访问
 */
@Module({
  providers: [PrismaService],
  exports: [PrismaService],
})
export class PrismaModule {}
```

## 注意事项

1. **数据备份**: 在进行数据库迁移前，请务必备份现有MySQL数据库
2. **依赖冲突**: 移除TypeORM相关依赖时，确保没有其他模块依赖这些包
3. **代码兼容性**: Prisma的API与TypeORM不同，需要重写数据访问层代码
4. **测试覆盖**: 确保所有测试用例都能通过新的数据库配置运行
5. **环境变量**: 在不同环境（开发、测试、生产）中使用不同的数据库配置
6. **性能优化**: 利用PostgreSQL的特性优化查询性能

通过以上配置，项目将正确使用PostgreSQL 16+和Prisma 5+作为数据库技术栈，符合文档要求。这将带来更好的性能、更丰富的功能特性以及现代化的开发体验。
