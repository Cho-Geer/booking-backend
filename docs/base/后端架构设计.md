## 概述
本文档详细描述预约系统的后端架构设计，包括技术栈选择、模块划分、数据访问层设计、API设计等方面。后端架构严格遵循项目需求文档中定义的技术要求，使用NestJS 10+作为主要框架，PostgreSQL 16+作为数据库，Prisma 5+作为ORM工具。

### 系统目标
- 提供稳定、高性能的后端服务支持
- 实现会议室预约、活动预约和个人服务预约等核心功能
- 支持用户认证授权、预约管理、通知系统等业务需求
- 确保数据安全和系统稳定性
- 提供完善的API文档和错误处理机制

### 技术栈要求
- **后端框架**: NestJS 10+ (使用最新LTS版本)
- **数据库**: PostgreSQL 16+ (最新稳定版本)
- **ORM**: Prisma 5+ (现代化ORM解决方案)
- **缓存**: Redis 7+ (最新版本)
- **身份验证**: JWT + Passport + bcrypt（JWT + Refresh Token机制）
- **API文档**: Swagger/OpenAPI
- **测试框架**: Jest 29+ + TestContainers (集成测试)

## 架构设计原则
1. **模块化设计**: 采用NestJS模块化架构，确保功能模块职责清晰、易于维护
2. **分层架构**: 严格分层（Controller层、Service层、Repository层），提高代码可维护性
3. **数据安全**: 敏感数据加密存储，HTTPS传输，完善的认证授权机制
4. **可扩展性**: 支持水平扩展，通过微服务架构支持未来业务扩展
5. **可观测性**: 完善的日志记录、监控和错误处理机制
6. **性能优化**: 合理使用缓存、数据库索引优化、连接池管理等技术

## 模块设计

### 1. 认证模块 (AuthModule)
负责用户认证和授权功能。

#### 核心功能
- 用户注册、登录、登出
- JWT Token生成和验证
- 密码重置
- 权限验证

#### 主要组件
- **AuthController**: 处理认证相关的HTTP请求
- **AuthService**: 实现认证业务逻辑
- **JwtStrategy**: JWT验证策略
- **LocalStrategy**: 本地验证策略

#### API接口
- `POST /auth/register`: 用户注册
- `POST /auth/login`: 用户登录
- `POST /auth/logout`: 用户登出
- `POST /auth/refresh`: 刷新访问令牌
- `POST /auth/forgot-password`: 忘记密码
- `POST /auth/reset-password`: 重置密码

### 2. 用户模块 (UsersModule)
负责用户信息管理功能。

#### 核心功能
- 用户信息CRUD操作
- 用户会话管理
- 用户权限管理
- 用户统计信息

#### 主要组件
- **UsersController**: 处理用户相关的HTTP请求
- **UsersService**: 实现用户业务逻辑
- **UsersRepository**: 用户数据访问层

#### API接口
- `GET /users`: 获取用户列表（管理员）
- `GET /users/:id`: 获取用户详情
- `PATCH /users/:id`: 更新用户信息
- `DELETE /users/:id`: 删除用户（管理员）
- `GET /users/profile`: 获取当前用户信息
- `PATCH /users/profile`: 更新当前用户信息

### 3. 预约模块 (BookingsModule)
负责预约管理功能，是系统的核心模块。

#### 核心功能
- 预约创建、查询、修改、取消
- 时间段管理
- 预约冲突检测
- 预约通知发送

#### 主要组件
- **BookingsController**: 处理预约相关的HTTP请求
- **BookingsService**: 实现预约业务逻辑

- `POST /bookings`: 创建预约

- `PATCH /bookings/:id`: 更新预约信息

### Repository模式
每个业务模块都有对应的Repository类，封装数据访问逻辑。

#### 示例：用户Repository
```typescript
// src/modules/users/users.repository.ts
import { Injectable } from '@nestjs/common';
import { PrismaService } from '../prisma/prisma.service';
import { CreateUserDto, UpdateUserDto } from './dto';

@Injectable()
export class UsersRepository {
  constructor(private prisma: PrismaService) {}

  async create(createUserDto: CreateUserDto) {
    return this.prisma.user.create({
      data: createUserDto,
    });
  }

  async findAll() {
    return this.prisma.user.findMany();
  }

  async findById(id: string) {
    return this.prisma.user.findUnique({
      where: { id },
    });
  }

  async findByPhone(phone: string) {
    return this.prisma.user.findUnique({
      where: { phone },
    });
  }

  async update(id: string, updateUserDto: UpdateUserDto) {
    return this.prisma.user.update({
      where: { id },
      data: updateUserDto,
    });
  }

  async delete(id: string) {
    return this.prisma.user.delete({
      where: { id },
    });
  }
}
```

### 错误处理
使用NestJS全局异常过滤器统一处理错误：

```typescript
// src/common/filters/http-exception.filter.ts
import { ExceptionFilter, Catch, ArgumentsHost, HttpException } from '@nestjs/common';
import { Response } from 'express';

@Catch(HttpException)
export class HttpExceptionFilter implements ExceptionFilter {
  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse<Response>();
    const status = exception.getStatus();
    const message = exception.message || 'Internal server error';

    response.status(status).json({
      success: false,
      error: {
        code: status.toString(),
        message,
      },
    });
  }
}
```

## 安全设计

### 认证机制
1. **JWT Token**: 使用JWT进行无状态认证
2. **Refresh Token**: 支持刷新令牌，提高安全性
3. **密码加密**: 使用bcrypt进行密码加密存储
4. **会话管理**: 使用Redis存储会话信息

### 授权机制
1. **RBAC**: 基于角色的访问控制
2. **权限验证**: 在路由级别和业务逻辑级别进行权限验证
3. **数据权限**: 确保用户只能访问自己的数据

### 数据安全
1. **敏感数据加密**: 用户手机号等敏感信息加密存储
2. **HTTPS传输**: 所有API请求使用HTTPS传输
3. **输入验证**: 使用DTO和验证管道确保输入数据安全
4. **SQL注入防护**: 使用Prisma ORM避免SQL注入风险

### 安全防护
1. **速率限制**: 防止API滥用
2. **CSRF防护**: 防止跨站请求伪造
3. **XSS防护**: 防止跨站脚本攻击
4. **日志审计**: 记录所有重要操作日志

## 性能优化

### 数据库优化
1. **索引优化**: 为常用查询字段创建索引
2. **查询优化**: 避免N+1查询问题
3. **连接池**: 合理配置数据库连接池
4. **分页查询**: 大数据量查询使用分页

### 缓存策略
1. **Redis缓存**: 使用Redis缓存热点数据
2. **查询缓存**: 缓存频繁查询的结果
3. **会话缓存**: 使用Redis存储用户会话

### API优化
1. **响应压缩**: 启用Gzip压缩减少响应体积
2. **CDN加速**: 静态资源使用CDN加速
3. **懒加载**: 合理使用懒加载减少初始加载时间

## 部署架构

### 容器化部署
使用Docker和Docker Compose进行容器化部署：

```yaml
# docker-compose.yml
version: '3.8'
services:
  backend:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://postgres:password@db:5432/booking_system
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db
      - redis

  db:
    image: postgres:16
    environment:
      - POSTGRES_DB=booking_system
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7
    ports:
      - "6379:6379"

volumes:
  postgres_data:
```
