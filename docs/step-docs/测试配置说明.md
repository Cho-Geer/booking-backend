# 测试配置说明

## 概述
本文档详细说明了预约系统的测试配置，包括测试技术栈、环境配置、测试用例设计和最佳实践。基于`/docs/base/`目录中的文档规范，确保测试配置与当前代码保持一致。

## 测试技术栈

### 核心测试框架
- **TestContainers 11.6.0+**: 提供真实的数据库容器环境
- **Jest 29.7.0+**: 主要的测试运行器和断言库
- **Supertest 6.3.4+**: HTTP请求测试库
- **@nestjs/testing 10.0.0+**: NestJS官方测试工具包

### 数据库测试
- **PostgreSQL 16+容器**: 使用TestContainers启动真实的PostgreSQL数据库
- **Prisma 6.16.2+**: 现代化的ORM，支持TypeScript和数据库迁移

### 测试类型
- **单元测试**: 测试单个函数或方法的逻辑
- **集成测试**: 测试模块间的集成和数据库操作
- **端到端测试**: 测试完整的API请求响应流程
- **性能测试**: 使用k6进行负载测试

## 后端测试配置

### TestContainers配置 (test/setup.ts)
```typescript
import { Test } from '@nestjs/testing';
import { PostgreSqlContainer } from '@testcontainers/postgresql';
import { PrismaService } from '../src/modules/prisma/prisma.service';

let postgresContainer: PostgreSqlContainer;

// 全局测试设置
beforeAll(async () => {
  // 启动PostgreSQL容器
  postgresContainer = await new PostgreSqlContainer()
    .withDatabase('test_booking_system')
    .withUsername('postgres')
    .withPassword('test_password')
    .start();

  // 设置环境变量
  process.env.DATABASE_URL = postgresContainer.getConnectionUri();

  // 创建测试模块
  const moduleRef = await Test.createTestingModule({
    imports: [AppModule],
  }).compile();

  const app = moduleRef.createNestApplication();
  await app.init();

  // 运行数据库迁移
  const prismaService = app.get(PrismaService);
  await prismaService.$executeRaw`CREATE EXTENSION IF NOT EXISTS "uuid-ossp";`;
  await prismaService.$executeRaw`CREATE EXTENSION IF NOT EXISTS "pgcrypto";`;
});

// 全局测试清理
afterAll(async () => {
  await postgresContainer.stop();
});
```

## 测试用例

### 用户服务测试 (users.service.spec.ts)
```typescript
describe('UsersService', () => {
  let service: UsersService;
  let prisma: PrismaService;

  beforeEach(async () => {
    const module: TestingModule = await Test.createTestingModule({
      providers: [
        UsersService,
        {
          provide: PrismaService,
          useValue: {
            user: {
              findUnique: jest.fn(),
              findFirst: jest.fn(),
              create: jest.fn(),
              update: jest.fn(),
              delete: jest.fn(),
              findMany: jest.fn(),
              count: jest.fn(),
            },
          },
        },
      ],
    }).compile();

    service = module.get<UsersService>(UsersService);
    prisma = module.get<PrismaService>(PrismaService);
  });

  describe('createUser', () => {
    it('应该成功创建用户', async () => {
      const createUserDto: CreateUserDto = {
        name: '测试用户',
        phone: '13800138000',
        email: 'test@example.com',
      };

      const mockUser = {
        id: 'uuid-123',
        ...createUserDto,
        phone: '138****8000',
        phoneHash: 'hashed-13800138000',
        status: 'active',
        createdAt: new Date(),
        updatedAt: new Date(),
      };

      jest.spyOn(prisma.user, 'findUnique').mockResolvedValue(null);
      jest.spyOn(prisma.user, 'create').mockResolvedValue(mockUser);

      const result = await service.createUser(createUserDto);
      expect(result).toEqual(mockUser);
      expect(prisma.user.create).toHaveBeenCalledWith({
        data: expect.objectContaining({
          name: '测试用户',
          phone: '138****8000',
          phoneHash: expect.any(String),
        }),
      });
    });

    it('手机号已被注册时应抛出异常', async () => {
      const createUserDto: CreateUserDto = {
        name: '测试用户',
        phone: '13800138000',
      };

      const existingUser = {
        id: 'uuid-123',
        name: '已存在用户',
        phone: '138****8000',
        phoneHash: 'hashed-13800138000',
      };

      jest.spyOn(prisma.user, 'findUnique').mockResolvedValue(existingUser);

      await expect(service.createUser(createUserDto)).rejects.toThrow(
        new PhoneNumberExistsException('手机号 13800138000 已存在'),
      );
    });
  });
});
```

## 测试最佳实践

### 1. 测试隔离
- 每个测试用例应该独立运行，不依赖其他测试的状态
- 使用beforeEach/afterEach清理测试数据
- 避免测试用例间的依赖和状态共享

### 2. 测试数据管理
- 使用工厂函数创建测试数据（如`createUserFactory`）
- 测试数据应该具有代表性，覆盖各种边界情况
- 测试完成后清理测试数据以避免污染
- 使用TestContainers确保数据库状态隔离

### 3. 错误处理测试
- 测试正常的成功路径（happy path）
- 测试异常的错误路径（error path）
- 验证错误消息和状态码的正确性
- 测试边界条件和异常输入

### 4. 性能考虑
- 避免在测试中使用真实的外部服务（邮件、短信等）
- 使用模拟（mocks）和存根（stubs）替代真实调用
- 控制测试运行时间，避免长时间运行的测试
- 使用Jest的异步测试支持

### 5. 测试覆盖率
- 业务逻辑层覆盖率 > 90%
- 控制器层覆盖率 > 85% 
- 服务层覆盖率 > 95%
- 工具函数覆盖率 100%
- 异常处理覆盖率 100%

## 持续集成配置

### GitHub Actions (.github/workflows/test.yml)
```yaml
name: Test

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: postgres
          POSTGRES_PASSWORD: postgres
          POSTGRES_DB: test_booking_system
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Setup Node.js
      uses: actions/setup-node@v4
      with:
        node-version: '20'
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
    
    - name: Run tests
      env:
        DATABASE_URL: postgresql://postgres:postgres@localhost:5432/test_booking_system
        NODE_ENV: test
        JWT_SECRET: test-jwt-secret-key-for-testing-only
      run: npm test
    
    - name: Upload coverage reports
      uses: codecov/codecov-action@v4
      with:
        file: ./coverage/lcov.info
```

## 故障排除

### 常见问题及解决方案

1. **TestContainers启动失败**
   ```bash
   # 检查Docker是否运行
   docker info
   
   # 设置TestContainers配置
   export TESTCONTAINERS_RYUK_DISABLED=true
   export TESTCONTAINERS_HOST_OVERRIDE=localhost
   ```

2. **数据库连接问题**
   ```bash
   # 检查数据库连接
   psql -h localhost -p 5432 -U postgres -d test_booking_system
   
   # 检查环境变量
   echo $DATABASE_URL
   ```

3. **Jest超时问题**
   ```javascript
   // jest.config.js
   testTimeout: 30000 // 增加超时时间
   ```

4. **Prisma连接问题**
   ```bash
   # 重新生成Prisma客户端
   npm run prisma:generate
   
   # 运行数据库迁移
   npm run prisma:migrate
   ```

5. **内存不足问题**
   ```bash
   # 增加Node.js内存限制
   export NODE_OPTIONS="--max-old-space-size=4096"
   ```

## 注意事项

1. **环境变量**: 确保测试环境变量正确设置，特别是数据库连接字符串
2. **数据库清理**: 测试前后清理数据库，确保测试隔离性
3. **网络连接**: 确保Docker和TestContainers正常工作，网络连接稳定
4. **资源限制**: 注意测试容器的资源使用，避免资源耗尽
5. **调试支持**: 使用Jest的调试模式进行问题排查
6. **版本兼容**: 确保测试依赖版本与项目依赖版本兼容
